# Phase 2 Briefs

Execution briefs and architectural planning documents for Phase 2.
Phase 2 begins after the Phase 1.3 Reality Check produces its triage
output; this folder holds the forward-looking planning material that
will inform the Phase 2 Execution Brief when it is written.

What goes here: architectural briefs for Phase 2 work that is
specified but not yet built. Each file describes a Phase 2 obligation
with enough structure that the Phase 2 brief writer can turn it into
concrete tasks without rediscovering the design.

What does NOT go here: Phase 1.1 or Phase 1.2 content (see their
respective sibling folders), specs for rules enforced in code today
(see `docs/02_specs/`), or the Phase 2 Execution Brief itself (which
will be written after Phase 1.3 and live here as `brief.md` when it
lands).

AI agents may create new files in this folder during Phase 2 scoping
work, but `interaction_model_extraction.md` is a human-authored
architectural statement witnessed and preserved from the commit-4b
prelude and should not be modified without explicit human approval.
